The ER diagram represents the core entities involved in the Weather Prediction System and illustrates how they interact with each other. It provides a clear structure for organizing User data, Weather data, ML Models, and Predictions.

## ER Diagram Visualization

## Main Entities

### 1. User
Represents the system user (admin, meteorologist, disaster management officer, or local authority) who enters weather data and accesses prediction results.

**Attributes:** User ID, User Name, Email, Password, Role

### 2. Weather Data
Stores environmental and rainfall-related parameters used for flood prediction. Each record is associated with a specific user.

**Attributes:** Data ID, User ID, Annual Rainfall, Cloud Visibility, Seasonal Rainfall, Temperature, Humidity

### 3. Machine Learning Model
Contains information about the trained prediction model, including algorithm details and performance metrics.

**Attributes:** Model ID, Model Name, Algorithm Type, Accuracy, Model File

### 4. Prediction
Stores the flood prediction outcome generated by the machine learning model, along with probability and prediction date.

**Attributes:** Prediction ID, Data ID, Model ID, Prediction Result, Flood Probability, Prediction Date

## Relationships

- **User → Weather Data:** One user can submit multiple weather records (One-to-Many).
- **Weather Data → Prediction:** Each weather record generates one prediction result (One-to-One).
- **Machine Learning Model → Prediction:** One model can generate multiple prediction results (One-to-Many).

This ER model supports efficient management of weather inputs, machine learning models, and flood prediction records.
